# Ph.3 設計・品質・チーム開発

## ゴール

**このフェーズが「未経験者と現場経験者の差」が最も大きい領域です。**

コードを書くだけでなく、「なぜこう設計したか」を言語化できるレベルを目指します。
加えて、「自分が書いていないコードを読んで改善できる」力を鍛えます。

## 期間の目安

**2〜3ヶ月**

## フェーズ通過基準

- DB設計のトレードオフを説明できる
- Mockitoで依存クラスをモックしたテストが書ける
- SOLID原則の各項目を具体例で説明できる
- **「読めないコード」を読んで何をしているか説明できる**
- **他人のコードにレビューコメントをつけられる（理由つき）**

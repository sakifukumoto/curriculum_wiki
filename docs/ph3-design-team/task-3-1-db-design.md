# 課題3-1：DB設計・ER図作成

## この課題のゴール

要件からDB設計を自分で行えるようになる。正規化の概念を理解する。

## 目安時間

15〜20時間

## 仕様

以下の要件からDB設計を行ってください。

### 要件

チームのタスク管理ツールを作ります。

- ユーザーはチームに所属できる（1人が複数チームに所属可能）
- チームはプロジェクトを持つ
- プロジェクトはタスクを持つ
- タスクはユーザーにアサインできる（担当者は1人）
- タスクにはコメントをつけられる
- タスクにはラベルをつけられる（1タスクに複数ラベル可能）

### 成果物

1. **ER図**（draw.io や Figma などで作成）
2. **DDL**（CREATE TABLE文）
3. **設計の説明文**（なぜこの設計にしたかの説明）

### 設計時に考慮すること

- 正規化（第1〜第3正規形）
- 適切なインデックスの設計
- NULL許容・NOT NULL制約の判断

## OK基準

- [ ] 要件を全て満たすテーブル設計になっている
- [ ] 多対多の関係が中間テーブルで表現されている
- [ ] 主キー・外部キーが適切に設定されている
- [ ] よく検索するカラムにインデックスが設定されている
- [ ] PR本文に「設計で迷った点と採用した理由」が書かれている

# 課題の進め方・提出方法

## 課題ページの構成

各課題ページは以下の構成になっています。

```
## この課題のゴール   → 何を目的としているか
## 目安時間          → どのくらいかかるか
## 仕様              → 何を作るか
## 成果物            → 何を提出するか
## OK基準            → 何を満たせばOKか
## 参考              → 参考になるリンク
```

**「OK基準」が最重要です。** 実装を始める前に必ず読んで、何を満たせばOKかを理解してから始めてください。

---

## 課題の進め方（ステップ別）

### Step 1：課題ページを読む（15〜30分）

1. 課題ページを最初から最後まで全部読む
2. OK基準を読んで「何を作ればいいか」を理解する
3. わからない用語をざっと調べる（深追いしない）

### Step 2：ブランチを切る

```bash
# mainを最新にしてから
git pull origin main

# featureブランチを切る
git checkout -b feature/[自分の名前]/[フェーズ]-[課題番号]-[内容]

# 例
git checkout -b feature/yamada/ph1-task-1-1-oop
```

### Step 3：実装する

- 少しずつ実装してこまめにcommitする
- 「1機能 = 1コミット」を意識する
- 詰まったら [困ったときの対処法](05-when-stuck.md) を参照

### Step 4：OK基準を全てチェックする

実装が終わったら、課題ページのOK基準を1つずつ確認します。

- [ ] 全て満たしているか？
- [ ] 動作確認をしたか？

### Step 5：PRを出す

```bash
# GitHubにpush
git push origin feature/[自分の名前]/[課題番号]
```

GitHubを開いて「Compare & pull request」からPRを作成します。  
PRテンプレートの**3項目を全て記入**してから提出します。

### Step 6：レビューを受ける

- 講師がPRにコメントをつけます（通常1〜3日以内）
- NGの場合は修正してpushし、返信を書きます
- OKが出たら次の課題に進みます

## 成果物の置き場所

**自分専用のディレクトリに必ず置くこと。**

```
submissions/
└── [自分の名前]/       ← ここだけ触る
    ├── ph0/
    │   ├── hello.txt
    │   └── sql-practice.sql
    ├── ph1/
    │   └── task-1-1/
    │       ├── Book.java
    │       ├── EBook.java
    │       └── Library.java
    └── ph2/
        └── task-2-2/
            └── （Spring Bootプロジェクト一式）
```

> ⚠️ **他のメンバーのディレクトリには絶対に触らないこと。**  
> 触るとコンフリクトの原因になります。

## 進捗の目安

| 状況 | 次の行動 |
|------|---------|
| 順調に進んでいる | そのまま続ける |
| 詰まって30分経った | [困ったときの対処法](05-when-stuck.md) を読む |
| 詰まって1日経った | 講師に質問する |
| 目安時間の2倍かかっている | 講師に相談する |
| フェーズの切り替わり | 講師に報告する |

## よくある間違い

### mainブランチに直接コミットしてしまった

```bash
# 直前のコミットを取り消す
git reset --soft HEAD~1

# featureブランチを切り直す
git checkout -b feature/yamada/ph1-task-1-1-oop

# もう一度コミット
git add .
git commit -m "feat: Bookクラスを実装"
```

### 間違えて他の人のファイルを変更してしまった

```bash
# 変更を確認
git status

# 変更を元に戻す（そのファイルだけ）
git checkout -- submissions/他の人の名前/...
```

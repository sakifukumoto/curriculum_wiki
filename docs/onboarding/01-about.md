# このカリキュラムについて

## 目標

このカリキュラムを修了すると、**Java / Spring Boot を使った現場での開発に参加できるレベル**を目指します。

具体的には以下ができる状態です。

- Spring BootでREST APIを設計・実装・テスト・デプロイできる
- GitとGitHubを使ったチーム開発フローで動ける
- エラーや仕様不明点を自分で調べて解決できる
- 設計の意図を言葉で説明できる

## このカリキュラムの進め方

### 「投げやり」は意図的な設計です

課題には仕様とOK基準だけが書かれており、「どうやって実装するか」は書いていません。

これは意図的な設計です。現場では「このAPIを実装してください」という仕様書が渡されるだけで、実装方法は自分で調べます。**その力を今から鍛えます。**

### ただし、詰まったら質問してください

自分で調べることは大事ですが、**30分調べてもわからない場合は講師に質問する**のがルールです。

「こんなことを聞いていいのか」と遠慮する必要はありません。どこで詰まっているかを具体的に伝えてください。

### FBを大切にする

講師のフィードバック（FB）は週1〜2時間という限られた時間の中で行われます。

**FBを最大限活かすために：**
- PR本文の「迷った点」欄に疑問を書いておく
- FBコメントへの返信は「修正しました」だけでなく「〇〇という理由で修正しました」と書く
- FBで指摘されたことは、次の課題でも意識する

## 心構え

### 「動いた」は半分のゴール

コードが動くことは必要条件ですが十分条件ではありません。  
「なぜこう書いたか」を説明できて初めてOKです。

### エラーは敵ではない

エラーメッセージはヒントです。怖がらずに読みましょう。  
詳しくは [エラーの読み方ガイド](../ph0-foundation/how-to-debug.md) を参照してください。

### 進捗の遅れは早めに報告する

詰まって進めない状態が2〜3日続いたら、すぐに講師に連絡してください。  
遠慮して黙っていると、気づいたときには大きく遅れていることがあります。

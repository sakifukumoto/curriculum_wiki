# よくある質問

## カリキュラムについて

**Q. プログラミングが全くわからなくても大丈夫ですか？**

大丈夫です。Ph.0は「PC操作が不慣れな人」を想定して設計されています。ただし、週20時間の学習時間を確保できることが前提です。

---

**Q. 目安時間より大幅にかかっています。おかしいですか？**

最初は目安時間の2〜3倍かかることはよくあります。ただし、目安時間の3倍以上かかっている場合は一度講師に相談してください。詰まっている箇所が別にある可能性があります。

---

**Q. 課題を飛ばして先に進んでもいいですか？**

原則NGです。各課題は後の課題の前提知識になっています。特にPh.0〜Ph.1はPh.2以降の基盤なので、しっかり取り組んでください。

不明点がある場合は「理解が浅いまま進むより、飛ばしたい理由を講師に相談する」のが正しい対処です。

---

**Q. FBはいつ来ますか？**

通常1〜3日以内です。急ぎの場合は講師に連絡してください。

FBを待っている間は次の課題に進むか、現在の課題のリファクタリングに取り組みましょう。

---

**Q. 自分なりに改良を加えてもいいですか？**

OKです。ただし、OK基準を全て満たした上で改良してください。OK基準を満たしていない状態での追加実装は評価されません。

---

## Gitについて

**Q. コミットはどのくらいの頻度でするべきですか？**

「1つの変更を実装したら1コミット」が目安です。1日1コミットでは少なすぎます。

例：
- ✅ Bookクラスを作成した → コミット
- ✅ LibraryクラスにfindByIdメソッドを追加した → コミット
- ❌ 全部実装し終わってから1コミット

---

**Q. コミットメッセージは日本語でいいですか？**

このカリキュラムでは日本語でOKです。ただし現場では英語が多いです。慣れてきたら英語で書く練習もしてみてください。

---

**Q. PRを出した後に修正したい場合はどうしますか？**

同じブランチに追加のコミットをpushするだけでOKです。PRは自動的に更新されます。

```bash
# 修正する
# ...

git add .
git commit -m "fix: レビュー指摘を修正"
git push origin feature/yamada/ph1-task-1-1-oop
```

## 技術について

**Q. IntelliJの使い方がわかりません**

[IntelliJ IDEA公式ドキュメント（日本語）](https://pleiades.io/help/idea/getting-started.html) が参考になります。

よく使うショートカットキーだけ覚えておくと作業効率が上がります。

| 操作 | Mac | Windows |
|------|-----|---------|
| 実行 | Cmd + Shift + F10 | Ctrl + Shift + F10 |
| 検索 | Cmd + F | Ctrl + F |
| 全検索 | Shift + Shift | Shift + Shift |
| フォーマット | Cmd + Option + L | Ctrl + Alt + L |
| インポート追加 | Option + Enter | Alt + Enter |

---

**Q. MySQLの接続でエラーが出ます**

よくある原因と解決法：

```bash
# MySQLが起動しているか確認（Mac）
brew services list | grep mysql

# 起動していない場合
brew services start mysql

# パスワードエラーの場合
mysql -u root -p
# パスワードを求められたら設定したパスワードを入力
```

**Q. Spring Bootのプロジェクトはどうやって作りますか？**

[Spring Initializr](https://start.spring.io/) を使います。

設定の目安（Ph.2から使用）：

| 項目 | 設定値 |
|------|--------|
| Project | Maven |
| Language | Java |
| Spring Boot | 3.x.x（最新安定版） |
| Java | 21 |
| Dependencies | Spring Web, Spring Data JPA, MySQL Driver, Validation |

## テーマA：タスク管理アプリ（推奨）

**難易度：★★☆（普通）**

Ph.2の課題の延長なので技術的にスムーズに進められます。

### 機能要件

**Must（必ず実装する）**

- ユーザー登録・ログイン（Spring Security）
- タスクの作成・編集・削除
- タスクにステータスを設定できる（TODO / IN_PROGRESS / DONE）
- タスク一覧画面（Thymeleaf）
- タスク作成・編集フォーム画面（Thymeleaf）

**Should（時間があれば）**

- タスクに期限を設定できる
- ステータスで絞り込める
- タスクにラベルを付けられる

**Won't（今回はやらない）**

- メール通知
- チーム共有機能

### 想定テーブル

`users` / `tasks` / `labels`（任意）

### 画面一覧

| 画面 | URL | 説明 |
|------|-----|------|
| ログイン | `GET /login` | ログインフォーム（Spring Securityが自動生成） |
| タスク一覧 | `GET /tasks` | タスクの一覧表示 |
| タスク作成 | `GET /tasks/new` | 作成フォーム |
| タスク編集 | `GET /tasks/{id}/edit` | 編集フォーム |

## テーマB：社内勤怠管理システム

**難易度：★★★（やや難しい）**

業務系システムの典型構成です。承認フローや集計処理が現場に近い経験になります。

### 機能要件

**Must（必ず実装する）**

- ユーザー登録・ログイン
- 権限管理（一般社員 / 管理者）
- 出勤・退勤の打刻
- 打刻履歴の一覧表示（Thymeleaf）
- 有給申請フォーム（Thymeleaf）

**Should（時間があれば）**

- 管理者が申請を承認・却下できる
- 月次の勤務時間集計

**Won't（今回はやらない）**

- 給与計算
- シフト管理

### 想定テーブル

`users` / `attendances` / `leave_requests` / `leave_types`

## テーマC：飲食店予約システム

**難易度：★★★（やや難しい）**

「空き管理」「同時アクセス時の整合性」など設計力が鍛えられます。

### 機能要件

**Must（必ず実装する）**

- 店舗オーナーによる予約管理画面

**Won't（今回はやらない）**

### 想定テーブル

`users` / `restaurants` / `reservations` / `tables`

## フロントの方針（全テーマ共通）

このカリキュラムはバックエンド専業を目指すため、フロントは **Thymeleaf で最低限** とします。

### 原則

- 一覧画面・フォーム画面・ログイン画面の3種類だけ作れば十分
- CSSは最低限（Bootstrap CDNを1行追加する程度でOK）
- JavaScriptは原則使わない
- **デザインに時間をかけないこと。評価されるのはAPIの設計・品質・テストです。**

### CSSを整えるには（任意）

HTMLの `<head>` に1行追加するだけです。

```html
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css" rel="stylesheet">
```

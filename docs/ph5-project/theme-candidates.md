# テーマ候補一覧

以下の3つから1つ選んでください。自分でテーマを提案することも可能です（講師と相談）。

---

## テーマA：タスク管理アプリ（推奨）

**難易度：★★☆（普通）**

Ph.2の課題の延長なので技術的にスムーズに進められます。初めての模擬プロジェクトに最適です。

### 機能要件

**ユーザー管理**
- ユーザー登録・ログイン（JWT認証）
- プロフィール表示・編集

**タスク管理**
- タスクの作成・編集・削除
- タスクにステータスを設定できる（TODO / IN_PROGRESS / DONE）
- タスクに期限を設定できる
- タスクを期限・ステータスで絞り込める

**ラベル機能**
- タスクにラベルを付けられる（1タスクに複数ラベル可）
- ラベルで絞り込める

**通知機能（任意）**
- 期限の3日前になったらメール通知

### 想定テーブル

`users` / `tasks` / `labels` / `task_labels`（中間テーブル）

## テーマB：社内勤怠管理システム

**難易度：★★★（やや難しい）**

業務系システムの典型構成です。承認フローや集計処理が含まれるため、現場での実務に近い経験ができます。

### 機能要件

**ユーザー管理**
- ユーザー登録・ログイン
- 権限管理（一般社員 / 管理者）

**勤怠管理**
- 出勤・退勤の打刻
- 打刻履歴の表示
- 有給申請・取得

**承認フロー**
- 管理者が申請を承認・却下できる
- 申請ステータスの管理（申請中 / 承認済み / 却下）

**集計・レポート**
- 月次の勤務時間集計
- 有給残日数の計算

### 想定テーブル

`users` / `attendances` / `leave_requests` / `leave_types`

---

## テーマC：飲食店予約システム

**難易度：★★★（やや難しい）**

「空き管理」「同時アクセス時の整合性」など、実装で考えることが多く設計力が鍛えられます。

### 機能要件

**ユーザー管理**
- ユーザー登録・ログイン
- 権限管理（一般ユーザー / 店舗オーナー）

**店舗管理（オーナー向け）**
- 店舗情報の登録・編集
- 席数・営業時間の設定
- 予約の確認・管理

**予約管理（ユーザー向け）**
- 空き状況の検索（日時・人数で検索）
- 予約の作成・キャンセル
- 予約履歴の確認

**通知機能（任意）**
- 予約確定メール
- 前日リマインドメール

### 想定テーブル

`users` / `restaurants` / `reservations` / `tables`

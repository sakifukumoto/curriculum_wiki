# Ph.5 実務模擬プロジェクト

## ゴール

**要件定義からデプロイまでを通しで体験する。**

このフェーズを終えると、現場に入った初日から「何をすべきか」がわかる状態になります。

## 期間の目安

**2ヶ月**（週20時間 × 8週 = 160時間）

## 進め方

### Week 1〜2：要件定義・設計

- [ ] 要件定義書を作成する
- [ ] ER図を作成する
- [ ] API仕様書（OpenAPI形式）を作成する
- [ ] GitHubのissueで全タスクを管理する

### Week 3〜6：実装

- [ ] Git Flowに沿って実装する
- [ ] PR単位で機能を追加する
- [ ] テストカバレッジ70%以上を維持する
- [ ] CI（GitHub Actions）でテストを自動実行する

### Week 7：デプロイ

- [ ] Dockerコンテナ化する
- [ ] AWSにデプロイする
- [ ] CDパイプラインを構築する

### Week 8：振り返り

- [ ] KPT（Keep / Problem / Try）で振り返りを行う
- [ ] 改善点をissueに起票する

## テーマ例

- タスク管理アプリ（Todoist風）
- 家計簿アプリ
- レストラン予約システム
- 社内勤怠管理システム

## 評価基準

| 観点 | 確認ポイント |
|------|------------|
| 設計力 | ER図・API仕様書が実装と一致しているか |
| 実装力 | レイヤー分離・例外処理・バリデーションが適切か |
| テスト | 主要な機能にテストが書かれているか |
| Git運用 | コミット・PR・レビューが適切に行われているか |
| 説明力 | 設計の意図をレビューで言語化できるか |

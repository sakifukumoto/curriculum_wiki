# Ph.5 実務模擬プロジェクト

## このフェーズの目的

**「現場に入った初日から動ける状態」を作ること。**

コードが書けることより、以下が身についているかが重要です。

- 要件を聞いて設計に落とせるか
- 設計の意図を言葉で説明できるか
- Git・PRを使ったチーム開発フローで動けるか
- 詰まったとき自分で調べて前に進めるか

## 前提条件

Ph.5を開始する前に、以下が全て完了していること。

- [ ] Ph.1〜Ph.4の全課題がApproved済み
- [ ] Spring BootでCRUD APIを一人で作れる
- [ ] Git FlowでPRを出してレビューを受けた経験がある
- [ ] Dockerで開発環境を立ち上げられる

## 進め方

| Week | フェーズ | 成果物 | 講師の関与 |
|------|---------|--------|-----------|
| 1 | テーマ選定・要件定義 | 要件定義書 | ⭐ 要件定義セッション（必須） |
| 2 | DB設計・API設計 | ER図・API仕様書 | ⭐ 設計レビュー（必須） |
| 3〜6 | 実装（スプリント×2） | 動くコード・PR履歴 | PRレビューのみ |
| 7 | デプロイ・結合テスト | 本番URL or Dockerで動くもの | 質問対応のみ |
| 8 | 振り返り・最終発表 | KPT・発表資料 | 最終FB |

## テーマ選択

[テーマ候補一覧](theme-candidates.md) から1つ選んでください。

自分でテーマを提案することも可能です。ただし以下の条件を満たすこと。

- Spring Boot + MySQL で実装できる
- CRUD操作が最低3リソース以上ある
- 認証（ログイン）機能を含む

# Ph.4 インフラ基礎知識

## このフェーズの方針

**目標：「使える・読める・困らない」レベル。「一から設計できる」は不要。**

現場のバックエンドエンジニアとインフラの関わり方は以下の通りです。

| やること | やらないこと |
|---------|------------|
| Dockerで立ち上がっている環境で開発する | DockerfileやComposeを一から設計する |
| CIが落ちたらログを読んで原因を特定する | CIパイプラインをゼロから構築する |
| AWSのサービス名と役割を理解している | インフラ構成を自分で設計・構築する |
| 本番のログ・エラーを読んで調査できる | サーバー障害をインフラレベルで対応する |

> インフラの専門的な構築はインフラエンジニア・SREの仕事です。
> バックエンドエンジニアは「インフラを理解した上でアプリを作る」ことが求められます。

---

## ゴール

- 既存のDockerfileを読んで、何をしているか説明できる
- `docker compose up` でローカル開発環境を立ち上げられる
- CIが落ちたときにログを読んで原因を特定できる
- AWS主要サービス（EC2・RDS・S3・ECS）の役割を説明できる
- 本番環境のログからエラーを調査できる

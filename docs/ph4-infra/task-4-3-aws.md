# 課題4-3：AWS基礎知識

## この課題のゴール

**「AWSを自分で構築できる」ではなく「AWSで動いているシステムを理解できる」レベルを目指します。**

現場では既にAWSインフラが構築されています。
あなたがやることは「どのサービスが何をしているか理解した上でアプリを開発する」ことです。

---

## 目安時間

8〜10時間（実装より知識確認中心）

---

## Step 1：主要サービスを理解する

以下の7サービスの役割を理解してください。
それぞれ「一言で言うと何か」を自分の言葉で説明できるレベルが目標です。

| サービス | 一言で言うと | Spring Bootとの関係 |
|---------|------------|-------------------|
| **EC2** | 仮想サーバー | Spring Bootアプリを動かすサーバー |
| **RDS** | マネージドDB | MySQLをAWSが管理してくれるサービス |
| **S3** | ファイルストレージ | 画像・ファイルの保存先 |
| **ECS** | コンテナ実行環境 | DockerコンテナをAWS上で動かす |
| **ECR** | コンテナレジストリ | DockerイメージをAWSで管理 |
| **ALB** | ロードバランサー | リクエストを複数サーバーに振り分ける |
| **VPC** | 仮想ネットワーク | AWSリソースをネットワークで隔離する |

### 現場でよく見るシステム構成

```
インターネット
     ↓
  ALB（ロードバランサー）
     ↓
  ECS（Dockerコンテナ）← Spring Bootアプリ
     ↓
  RDS（MySQL）

  ※ファイルはS3に保存
  ※DockerイメージはECRで管理
  ※全体がVPCの中に入っている
```

## Step 2：理解を確認する

以下の質問に自分の言葉で答えてください。
答えはPR本文の「Step 2 回答」欄に書いてください。

**Q1.** EC2とECSの違いを説明してください。
どちらも「アプリを動かす」ために使いますが、何が違いますか？

**Q2.** RDSを使う理由を説明してください。
EC2の中でMySQLを動かすのではなくRDSを使うのはなぜですか？

**Q3.** ALBがない構成とある構成の違いを説明してください。
なぜロードバランサーが必要なのですか？

**Q4.** 以下の現場での会話を読んで、何が起きているか説明してください。

```
先輩：「本番のECSタスクがクラッシュしてるから、CloudWatchでログ確認して」
自分：「わかりました。RDSへの接続エラーが出てます」
先輩：「RDSのセキュリティグループ、ECSからのインバウンドが許可されてるか確認して」
```

（ヒント：セキュリティグループ・CloudWatchで検索してみてください）

## Step 3：AWSコンソールを触ってみる（任意）

> ⚠️ **AWSは費用が発生します。** 無料枠の範囲で操作し、使い終わったら必ず削除してください。
> 不安な場合はこのStepはスキップしてOKです。

無料枠でできる最小限の確認：

1. AWSアカウントを作成する（無料）
2. S3バケットを作成してファイルをアップロード・ダウンロードする
3. 確認後すぐにバケットを削除する

S3だけで十分です。EC2・RDSは費用が発生しやすいので触らなくてOKです。

## このフェーズで「やらなくていいこと」

以下は**インフラエンジニア・SREの仕事**です。今は覚えなくて大丈夫です。

- ECSのタスク定義・サービスの設定
- RDSのパラメータグループ・バックアップ設定
- CloudFormation・TerraformなどIaCツール

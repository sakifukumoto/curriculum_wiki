# 課題2-5：Spring Security基礎

## この課題のゴール

JWT認証を使ったAPIの保護を実装できるようになる。

## 目安時間

20〜25時間

## 仕様

課題2-2のタスク管理APIに認証機能を追加してください。

### 追加するエンドポイント

| メソッド | パス | 認証 | 説明 |
|--------|------|------|------|
| POST | `/api/auth/register` | 不要 | ユーザー登録 |
| POST | `/api/auth/login` | 不要 | ログイン → JWTトークン返却 |
| GET | `/api/tasks` | 必要 | 自分のタスクのみ取得 |

### 認証フロー

```
1. POST /api/auth/login にID/PWを送信
2. サーバーがJWTトークンを返す
3. 以降のリクエストはHeaderに Bearer トークン をつける
4. サーバーはトークンを検証し、正当であればリクエストを処理
```

## OK基準

- [ ] ログインAPIがJWTトークンを返す
- [ ] トークンなしでタスクAPIを叩くと401が返る
- [ ] トークンありでタスクAPIを叩くと自分のタスクのみ返る
- [ ] パスワードが `BCrypt` でハッシュ化されている
- [ ] PR本文に「JWTの仕組みの説明」が書かれている

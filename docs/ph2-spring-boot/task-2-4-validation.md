# 課題2-4：バリデーション・エラー処理

## この課題のゴール

適切なバリデーションとエラーレスポンスを実装できるようになる。

## 目安時間

10〜15時間

## 仕様

課題2-2のタスク管理APIにバリデーションを追加してください。

### バリデーションルール

| フィールド | ルール |
|-----------|-------|
| `title` | 必須・1〜100文字 |
| `status` | `TODO` `IN_PROGRESS` `DONE` のいずれか |
| `description` | 500文字以内 |

### バリデーションエラーのレスポンス

```json
{
  "status": 400,
  "message": "入力値が不正です",
  "errors": [
    { "field": "title", "message": "タイトルは必須です" },
    { "field": "title", "message": "タイトルは100文字以内で入力してください" }
  ]
}
```

## OK基準

- [ ] `@Valid` アノテーションでバリデーションが動作する
- [ ] バリデーションエラーが400で返る
- [ ] エラーレスポンスにフィールドごとのメッセージが含まれる
- [ ] `@ControllerAdvice` でバリデーション例外をハンドリングしている
